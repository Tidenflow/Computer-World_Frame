关键需求：
1. 整个 CWFrameMap 呈现在用户面前
2. 用户不应该看到每个节点的 label
3. 用户通过输入框输入字符来匹配 node 的 label
4. 匹配成功后"点亮"对应节点
这是一个很有趣的设计挑战。让我分析一下：
Vue Flow 的适用性：
- Vue Flow 是一个流程图/节点图库，适合展示节点和边的关系
- 它支持自定义节点样式
- 支持边的动画等效果
但是，对于这个需求可能有更好的方案：
1. 纯 CSS/Canvas 方案：用 D3.js 或者纯 CSS 来做节点布局和点亮效果
2. CSS Grid/Flex + 动画：用 CSS 动画来实现"点亮"效果，可能更轻量
3. Vue Flow 的问题：
   - Vue Flow 是为"流程图"设计的，节点通常是可见的
   - 隐藏 label 但保留节点位置需要自定义节点
   - 可能有点"杀鸡用牛刀"
更简单的方案：
- 用 CSS 定位来展示整个框架（节点位置可以预设或自动计算）
- 节点以"点亮/未点亮"状态显示，初始时只展示形状或图标
- 输入框输入文字匹配成功后，通过 CSS 动画"点亮"对应节点
推荐方案：
用 CSS + 简单的节点定位来实现这个效果会更轻量、更好控制。如果需要更复杂的交互再考虑 Vue Flow。
可以尝试一个极简方案：用 CSS 绝对定位来布局节点，通过 filter: blur() 或 opacity 来控制可见性，输入匹配时用 CSS transition 实现点亮动画。这样的效果会更流畅，也更容易实现"隐藏 label"的探索感。
不过 Vue Flow 其实也能做到，支持自定义节点和边的样式。我应该给用户一个对比分析，帮助他根据团队熟悉度和具体需求来选择。
